I-3- Le neurone¶

La fonction qui met jour les paramtres prend en entre :

  • $W$ une liste des poids; un vecteur de taille $Lp$ (le nombre des neurones de la couche prcdente)
  • $b$ le biais
  • $z$ la combinaison linaire du neurone courant; un vecteur de taille $M$ (le nombre des chantillons)
  • $a$ l'activation du neurone courant; un vecteur de taille $M$
  • $a\_past$ les activations des neurones de la couche prcdente; une matrice de taille est $(M * Lp)$
  • $delta\_next$ le delta calcul dans la couche suivante; une matrice de taille $M * Ln$ ($Ln$ : le nombre des neurones dans la couche suivante)
  • $w\_next$ les poids vers la couche suivante; un vecteur de taille $Ln$
  • $act$ c'est un object de type "Activation"; il fournit deux mthodes : "act.activer" et "act.deriver"
  • $alpha$ le pas de l'entranement

def neurone_maj(W, b, z, a, a_past, delta_next, w_next, act, alpha=1.):
    delta = np.multiply(np.dot(delta_next,w_next), act.deriver(z,a))
    #dW = np.mean(np.dot(delta, a_past),axis=0)
    #db = np.mean(delta,axis=0)
    m = len(z)
    dW = (1/m) * np.dot(delta, a_past)
    db = (1/m) * np.sum(delta)
    W = W - alpha * dW
    b = b - alpha * db
    return W, b, delta

class Neurone(object):
    def __init__(self, taille_entree, activation=logistique):
        self.b = 0.
        self.w = np.array([0.] * taille_entree)
        self.activation = activation
        
    def randomiser(self):
        self.w = np.random.rand(len(self.w))
        self.b = np.random.rand(1)[0]
        
    def __aggreger(self, X):
        return np.dot(X, self.w) + self.b
    
    def activer(self, X):
        self.a_past = X
        self.z = self.__aggreger(X)
        self.a = self.activation.activer(self.z)
        return self.a
    
    def actualiser(self, delta_next, w_next, alpha=1.):
        w_ancien = self.w.copy()
        self.w, self.b, delta = neurone_maj(self.w, self.b, self.z, self.a, self.a_past, 
                                            delta_next, w_next, self.activation, alpha=alpha)
        return delta, w_ancien

class Couche(object):
    
    def __init__(self, taille, taille_entree, activation=logistique):
        self.neurones = [Neurone(taille_entree, activation=activation) for i in range(taille)]
        
    def randomiser(self):
        for neurone in self.neurones:
            neurone.randomiser()

    def propagation_avant(self, X):
        activations = []
        for neurone in self.neurones:
            activations.append(neurone.activer(X))
        return np.array(activations).T
    
    def retro_propagation(self, delta_next, W_next, alpha=1.):
        W_anciens = []
        Deltas = []
        for i, neurone in enumerate(self.neurones):
            delta, w_ancien = neurone.actualiser(delta_next, W_next[i], alpha=alpha)
            W_anciens.append(w_ancien)
            Deltas.append(delta)
        return np.array(Deltas).T, np.array(W_anciens).T

class RN(object):
    def __init__(self, taille_entree, cout=bce, alpha=1.):
        self.taille_courante = taille_entree #la taille de la dernire couche
        self.cout = cout #objet de type Cout pour calculer le cout et sa drive
        self.alpha = alpha
        self.couches = []

    def ajouter_couche(self, taille, activation=logistique):
        nouv_couche = Couche(taille, self.taille_courante, activation=activation)
        self.couches.append(nouv_couche)
        self.taille_courante = taille
        
    def randomiser(self):
        for couche in self.couches:
            couche.randomiser()
    
    def predire(self, X): 
        Y = X
        if self.norm:
            Y = np.where(self.std==0, X, (X - self.mean)/self.std)
            
        for couche in self.couches:
            Y = couche.propagation_avant(Y)
        if Y.ndim == 2 and Y.shape[1] == 1:
            Y = Y.flatten()
        return np.where(Y < 0.5, 0, 1)
    
    
    def _faire_iteration(self, X, Y):
        # propagation avant
        a = X
        for couche in self.couches:
            a = couche.propagation_avant(a)
            
        # calcul du cout et sa drive 
        YY = np.array(Y)
        if YY.ndim < 2 : 
            YY = YY[:, np.newaxis]
        J = np.mean(self.cout.calculer(a, YY))
        J_prime = self.cout.deriver(a, YY)
        
        # retropropagation 
        w_past = np.array([[1.] * self.taille_courante])
        delta_past = J_prime
#         l = 5
        for couche in reversed(self.couches): # on commance de la dernire couche vers la premire
#             print(l)
#             print(delta_past)
#             print(w_past)
#             l = l - 1
            delta_past, w_past = couche.retro_propagation(delta_past, w_past)
        return J
    
    def entrainer(self, X, Y, nbr_it=100, norm=False):
        couts = []
        X_norm = X
        self.norm = norm
        if norm:
            self.mean = np.mean(X, axis=0)
            self.std = np.std(X, axis=0)
            X_norm = np.where(self.std==0, X, (X - self.mean)/self.std)

        for i in range(nbr_it): 
            J = self._faire_iteration(X_norm, Y)
            couts.append(J)
        return couts

I-6-1- Randomisation des paramètres (thétas)¶

On va crer 4 modles :

  1. deux couches caches avec 4 et 2 neurones respectivement et des paramtres mis zro
  2. le mme que le prcdent mais on affecte des valeurs alatoires aux paramtres
  3. une seule couche cache avec 4 neurones et des paramtres mis zro

Analyse

On remarque que la fonction du cot ne change pas sa valeur lorsqu'on met les thtas zro, ce qui indique que les paramtres nne sont pas mis jour. Pourquoi? Pourtant, lorsqu'il y a une seule couche cache, les paramtres sont bien mis jour.

Rponse

En fait, toute initialisation constante fonctionnera trs mal. Considrons un rseau de neurones avec deux units caches et supposons que nous initialisons tous les $b$ 0 et les $W$ avec une constante $\alpha$. Si nous transmettons une entre $(x_{1}, x_{2})$ dans ce rseau, la sortie des deux units caches sera $activation\_function(\alpha x_{1},\alpha x_{2})$ Ainsi, les deux units caches auront une influence identique sur le cot, ce qui conduira des gradients identiques. Ainsi, les deux neurones volueront symtriquement tout au long de l'entrainement, empchant efficacement diffrents neurones d'apprendre des choses diffrentes.

I-6-2- Complexité du problème¶

On veut tester l'impact du nombre des couches (et des neurones) sur la convergence et le temps d'entranement. Pour ce faire, on a proposer 4 architectures avec une couche de sortie d'un seul neurone :

  1. 3 couches caches : 8 X 4 X 2 neurones
  2. 2 couches caches : 4 X 2 neurones
  3. 1 couche cache : 2 neurones
  4. 0 couches caches : pratiquement c'est de la rgression logistique

Analyse

  • Que remarquez-vous ?
  • Au del de 2 couches (cas 3 couches), on constate que le modle a arrt converger, comme le cas des paramtres initialiss zro. Pourquoi votre avis ?
  • Est-ce que ce problme est aussi complexe qu'on aura besoin de plusieurs couches caches ?
  • Dans ce problme, en comparant entre les deux architectures : 2 couches cachhes et la rgression logistique, quelle est la meilleure et pourquoi ?

Rponse

  • On remarque que la regression logistique converge plus rapidement que les rseaux de neuronnes d'une et deux couches, le rseau de neuronne de 3 couches s'arrete de converger plus rapidement que les autres.
  • les couches supplmentaires (au dela de 2)ont une influence identique sur le cot, ce qui conduira des gradients identiques. Ainsi, les neurones volueront symtriquement tout au long de l'entrainement, empchant efficacement diffrents neurones d'apprendre des choses diffrentes.
  • Non, le probleme n'est pas aussi complexe au point o une regression logistique est meilleure que les rseaux de neuronnes.
  • La regression logistique est meileure pour ce problme vu qu'elle minimise le cout mieux que le rseau de neuronne 2 couches et plus rapidement.

Ici, on essaye d'entrainer un modle qui reconnait les chiffres de 0 9. Il y a deux modles :

  • Dans le premire cellule, c'est une modle basique (
  • Dans la deuxime cellule, c'est un modle identique au premier. Sauf, vous devez le changer en ajoutant des couches : Dropout, MaxPooling2D, etc. Le deuxime modle doit tre une amlioration. Vous devez discuter comment vous l'avez amliorer, c--d, si vous avez ajouter une autre couche, pourquoi ?

Amlioration

Le modle a deux aspects principaux: le frontend d'extraction des features compos de couches convolutionnelles , et le backend classifieur qui fera une prdiction.

Pour le frontend convolutionnel, nous pouvons commencer avec deux couches convolutionnelles avec une petite taille de filtre (3,3) et un nombre modeste de filtres (32) suivie d'une couche de max pooling. Les mappes de filtres peuvent ensuite tre aplaties (Flatten) pour fournir les features au classifieur.

tant donn que le problme est une tche de classification multi-classes, nous savons que nous aurons besoin d'une couche de sortie 10 nuds afin de prdire la distribution de probabilit d'une image appartenant chacune des 10 classes. Cela ncessitera galement l'utilisation d'une fonction d'activation softmax. Entre l'extracteur de features et la couche de sortie, nous pouvons ajouter deux couches Dense pour interprter les features, dans ce cas avec 32 nuds chacune, avec un Dropout Rate de 0.5

Toutes les couches utiliseront la fonction d'activation ReLU et le schma d'initialisation du poids Xavier, les deux meilleures pratiques

L'algorithme d'optimization Adam est trs utilis dans les rseaux de neuronne. il combine le "momentum" et le "RMSprop". Le momentum aide la fonction du cout d'arriver au minimum global d'une faon plus rapide et consistante et cel en calculant la moyenne pondre des gradients chaque fois. RMSprop rendra la fonction de cot plus lente dans le sens vertical et plus rapide dans le sens horizontal.
